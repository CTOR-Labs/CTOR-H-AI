# UI Agent — Specification

## 1. Назначение

UI‑Agent — это автономный агент, отвечающий за анализ, улучшение и эволюцию пользовательского интерфейса платформы CTOR‑H‑AI.  
Он работает исключительно с визуальным слоем (HTML, CSS, JavaScript) и не затрагивает игровую логику или правила CTOR.

Основная цель агента — повышать удобство, понятность и эстетическое качество интерфейса, улучшая взаимодействие игрока с платформой.

---

## 2. Область ответственности

UI‑Agent выполняет следующие функции:

### 2.1. Анализ интерфейса
- Изучение структуры HTML.
- Анализ CSS‑стилей, выявление избыточности.
- Анализ JS‑кода, связанного с UI.
- Поиск проблем с адаптивностью.
- Проверка визуальной иерархии элементов.

### 2.2. Улучшение UX
- Предложения по улучшению читаемости.
- Подсветка активных элементов.
- Улучшение отображения ходов.
- Добавление визуальных подсказок.
- Оптимизация взаимодействия игрока с игрой.

### 2.3. Оптимизация кода
- Упрощение CSS.
- Рефакторинг HTML‑структуры.
- Оптимизация JS‑логики, связанной с UI.
- Удаление дублирующегося кода.

### 2.4. Доступность (Accessibility)
- Проверка контрастности.
- Размеры кликабельных элементов.
- Поддержка клавиатурной навигации.
- Улучшение доступности для слабовидящих пользователей.

### 2.5. Адаптивность
- Улучшение отображения на мобильных устройствах.
- Проверка поведения интерфейса при разных разрешениях.
- Предложения по адаптивной сетке.

---

## 3. Входные данные

UI‑Agent может работать со следующими типами данных:

- HTML‑файлы интерфейса.
- CSS‑файлы.
- JS‑файлы, связанные с визуальными элементами.
- Скриншоты интерфейса (опционально).
- Описание проблем от разработчика.
- История изменений UI.
- Ссылки на версии кода из `version-map.json`.

---

## 4. Выходные данные

Агент генерирует:

- Список улучшений UI.
- UX‑рекомендации.
- Оптимизированные фрагменты кода.
- Отчёты об анализе интерфейса.
- Предложения по адаптивности.
- Предложения по доступности.
- Рефакторинг‑планы.

## 5. Ограничения

UI‑Agent **не**:

- изменяет игровую логику;
- работает со стратегией ИИ;
- модифицирует правила CTOR;
- изменяет структуру движка CTOR‑Engine;
- принимает решения за Coordinator Agent.

---

## 6. Взаимодействие с другими агентами

UI‑Agent взаимодействует с:

### Game Logic Agent
- получает информацию о том, какие элементы UI должны отображать игровые состояния.

### AI‑Player Agent
- получает данные о том, какие визуальные подсказки могут помочь игроку понимать стратегию ИИ.

### Coordinator Agent (в будущем)
- получает задачи;
- отправляет отчёты;
- получает ссылки на версии кода.

## 7. Будущее расширение

- Автоматическое тестирование UI.
- Генерация UI‑прототипов.
- Интеграция с Deep Agents Workflow.

- Генерация визуальных тем (light/dark).
- Поддержка анимаций и переходов.
